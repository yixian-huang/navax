# nav.ax 产品需求全集

状态：`v1.1-frozen`
更新时间：2026-07-16
本文是产品范围与验收的权威来源；接口细节以 `api/openapi.yaml` 为准。

## 1. 产品定位

nav.ax 是一个开源、可自托管的个性化导航站。官方实例以 `nav.ax` 为主域名，提供系统公共导航、邀请制账号、个人导航编排、公开分享和运营管理。发行物必须同时支持原生单二进制和容器部署。

核心原则：打开即用、读多写少、默认保护隐私、单实例易维护、不过度引入外部服务。

## 2. 角色与权限

- **访客**：访问系统主页、`public` 或 `unlisted` 的个人分享页、发现页；不能读取草稿或私有数据。
- **用户**：管理自己的唯一导航页、分类、站点、布局、主题、发布状态、导入导出、会话及子域名申请。
- **管理员**：拥有用户全部能力，并管理系统主页、用户、邀请、公共目录、主题启用状态、系统配置、第三方服务、更新、备份和审计。

所有授权均由后端强制执行，前端路由守卫只负责体验。

## 3. 功能范围

### 3.1 初始化、认证与账号

- 首次启动进入初始化状态；使用启动时生成或由环境变量提供的 setup token 创建首个管理员。
- 首版仅允许邀请注册或完全关闭注册，不提供开放注册。
- 支持邮箱密码登录、退出、修改密码、查看并撤销活动会话、管理员撤销用户全部会话。
- 密码使用 Argon2id；认证使用服务端不透明 Session 和 HttpOnly Cookie。
- SMTP 未配置时可复制邀请链接；配置后可选择发送邀请邮件。

### 3.2 导航页、分类与站点

- 每个用户首版固定拥有一个个人导航页；系统主页使用同一 Page 模型并标记为 `system`。
- 支持分类与站点增删改、分类排序、站点排序及跨分类移动；排序操作必须事务化。
- 站点包含标题、HTTP(S) URL、图标、简介和排序值。URL 在同一页面内默认唯一。
- 删除非空分类时必须显式选择级联删除或移动到“未分类”，不允许静默丢失数据。
- 支持列表、紧凑网格、舒展网格，以及全功能、搜索聚焦、浏览优先、侧边栏四种首页布局。
- 首页提供搜索引擎切换、收藏过滤、时钟、日期和欢迎词。便签、天气和每日一言不进入 v1。

### 3.3 主题与资源

- 主题是随前端发布的受控包；管理员可启用、停用和设置默认主题，用户可选择启用主题。
- 用户偏好同步保存：主题、布局、密度、背景、时钟/日期/欢迎词显示状态。
- 图片默认可保存到本地数据目录；配置 S3 后可切换对象存储。远程图片 URL 必须通过协议和长度校验。
- v1 不支持上传或执行第三方主题代码。

### 3.4 草稿、发布与公开访问

- 编辑始终写入草稿；发布时生成不可变版本快照，后续编辑不影响已发布页面。
- 可见性为 `private | unlisted | public`：`private` 不可公开访问；`unlisted` 持链接可访问且 `noindex`；`public` 可进入发现页并允许索引。
- slug 在实例内全局唯一，使用小写字母、数字和连字符；保留系统路由词。
- 公开页只读取发布快照，不泄露邮箱、私有备注、草稿或管理字段。
- Go 服务端为公开页注入 title、description、canonical、robots 和 Open Graph 元数据。

### 3.5 nav.ax 域名与发现页

- `/` 是 nav.ax 系统导航主页，用户可见品牌统一使用 `nav.ax`；自部署实例可覆盖实例名称。
- 个人分享页默认地址为 `/u/{slug}`。
- 用户可申请 `{label}.nav.ax`：4 个及以上字符的可用、非保留名称自动启用；1–3 个字符作为稀缺短域名进入管理员审核。已启用名称仍可因违规被管理员撤销。
- v1 不接入支付；短域名由管理员人工授予。后续付费权益只控制短域名申请资格，不改变域名路由模型。
- 在通配 DNS/TLS 部署下只需应用内分配和路由。DNS Provider API 作为可选扩展配置。
- 发现页只展示 `public` 页面，支持关键词、标签、最新与热门排序；管理员可设置精选状态。

### 3.6 商业化边界（后续版本）

- 免费能力保持完整可用：标准链接额度、公开分享，以及 4 个及以上字符的 `nav.ax` 子域名。
- 个人高级权益聚焦更高的分类/链接额度和 1–3 个字符的稀缺短子域名。
- 品牌高级权益聚焦隐藏平台品牌、替换 Logo/名称/页脚和分享元信息等白标能力。
- 计费、订单、支付渠道和自动续费不进入 v1；引入支付前先落地统一的套餐、配额和权益判定层。

### 3.7 导入、导出与链接检查

- 支持浏览器书签 HTML 和版本化 nav.ax JSON 导入，提供 `merge | replace` 模式、预检结果和原子提交。
- 同一个 idempotency key 的导入不得重复写入。
- 用户可导出可迁移 JSON；管理员可创建完整实例备份。
- 批量链接检查限制数量、并发和超时，并阻止访问回环、私网、链路本地和云元数据地址。

### 3.8 访问统计

- 记录页面浏览和站点点击，提供按日 PV/UV、热门站点、分类占比和最近访问。
- 默认不持久化完整 IP；仅保存带每日盐的访客散列和粗粒度国家/设备信息。
- 管理员可关闭统计并配置数据保留天数；用户只能查看自己的页面统计。

### 3.9 运营管理

- 用户：分页筛选、启用/禁用、查看详情、撤销会话。
- 邀请：创建、复制、可选邮件发送、有效期、次数限制、撤销。
- 系统主页与公共目录：分类/站点 CRUD、排序、启停。
- 主题：启停、设置默认；系统设置：实例、注册、限额、背景、统计、域名策略。
- 审计：记录管理员及安全敏感操作，不记录密码、Session、Access Key 等秘密。

### 3.10 第三方服务配置

管理端提供 SMTP、S3 和 DNS Provider 的读取（秘密脱敏）、更新、测试和禁用 API。秘密使用环境主密钥加密后入库；没有主密钥时禁止持久化秘密。外部服务未配置不得阻塞核心导航功能。

### 3.11 自动更新与备份

- 定期检查带版本、平台资产、SHA-256 和 Ed25519 签名的发布清单。
- 管理端可查看当前版、可用版、检查时间和发行说明，并手动触发检查。
- 原生二进制部署支持下载、验签、更新前备份、原子替换和失败回滚；默认不自动应用。
- 容器部署只检查并提示新镜像，不在容器内自我替换。
- 支持管理员下载备份；恢复必须再次认证并提供一次性确认 token。

## 4. 非功能要求

- 单实例 SQLite WAL；所有写操作使用短事务，公开读取可缓存并支持 ETag。
- 同源 REST API 前缀 `/api/v1`；结构化 JSON 日志、request ID、`/healthz`、`/readyz` 和版本信息。
- 支持 Linux amd64/arm64；前端、迁移和默认资源嵌入 Go 二进制。
- 所有输入有长度、格式和权限校验；登录、邀请验证、公开统计和外部探测必须限流。
- WCAG AA、键盘操作、移动端适配；用户消息以中文为默认语言。

## 5. 明确不做

- 微服务、Redis、消息队列、多数据库兼容和多节点写入。
- 团队协作导航、任意插件执行、第三方主题代码、实时多人编辑。
- v1 内建完整邮件营销、付费订阅或任意自定义域名托管。

## 6. 上线验收

- 全新实例可完成初始化、邀请注册、登录、导航编辑、发布和公开访问。
- 草稿与发布快照隔离；权限、可见性、禁用账号和 Session 撤销符合规则。
- 管理员可完成用户、邀请、系统主页、目录、主题、设置、服务配置、更新检查和备份操作。
- 所有 OpenAPI 操作均有后端实现和契约测试；前端不再依赖业务 Mock 或服务端状态的 localStorage。
- Go 单元/集成测试、前端检查、关键 E2E、生产构建、二进制及容器冒烟测试全部通过。
